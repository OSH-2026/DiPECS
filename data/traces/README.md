# Trace Fixtures

This directory contains small golden fixtures and Android-shaped JSONL samples
for replay, dashboard, and policy/audit validation.

## Synthetic Android Large Sample

`android_synthetic_large.redacted.jsonl` is a deterministic synthetic Android
collector trace. It is generated by:

```bash
python tools/generate_synthetic_android_trace.py --rows 2400
```

The default generated sample has:

- 2400 total JSONL rows.
- 1830 rows with non-null Rust-compatible `rawEvent`.
- 570 `rawEvent: null` screening/internal rows.
- Coverage for `AppTransition`, `NotificationPosted`,
  `NotificationInteraction`, `ScreenState`, and `SystemState`.
- Android-safe action trace rows for prefetch, release memory, keepalive,
  own-resource prewarm, and user-visible action hints.

The companion summary is written to
`android_synthetic_large.redacted.summary.json`.

The sample is intentionally redacted:

- Notification `raw_title` and `raw_text` are empty.
- Accessibility text and content descriptions are null.
- Action socket tokens, cache paths, and raw action targets are null.
- Each row has `synthetic: true` and `redacted: true`.

Do not report this file as real-device evidence. Use it when there is no time
to collect a large true-device trace, or when a reproducible fixture is needed
for local dashboard/replay demos.

Replay example:

```bash
cargo run -p aios-cli -- replay data/traces/android_synthetic_large.redacted.jsonl \
  --stages policy \
  --audit data/evaluation/android_synthetic_large.audit.ndjson
```
