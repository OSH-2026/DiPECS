# Lab4 Qwen3.5 输出质量评估报告

## 1. 实验目标

使用固定的 5 条质量 prompt，对 Qwen3.5-2B Q4_K_M 在三种温度配置下的输出进行
对比，观察采样温度对响应速度、指令遵循、完整性和事实准确性的影响。

## 2. 实验配置

除温度外，三组配置保持一致：

| 项目 | 内容 |
| :--- | :--- |
| 模型 | Qwen3.5-2B Q4_K_M |
| 后端 | llama.cpp CPU |
| 线程数 | 12 |
| 上下文 | 1024 |
| Batch | 64 |
| Seed | 42 |
| Reasoning | off |
| 对比变量 | `temperature=0.2`、`temperature=0.5`、`temperature=0.8` |

Prompt 覆盖中文问答、摘要、Rust 代码解释、RPC 原因分析和 OS 知识题共 5 类。

## 3. 性能结果

| 配置 | 成功数 | 平均端到端耗时 | 平均生成速度 |
| :--- | ---: | ---: | ---: |
| temperature=0.2 | 5/5 | 8,868.8 ms | 31.58 t/s |
| **temperature=0.5** | **5/5** | **8,436.4 ms** | **32.12 t/s** |
| temperature=0.8 | 5/5 | 9,454.2 ms | 28.90 t/s |

**temperature=0.5** 的平均耗时最短、生成速度最快，但三组差异主要源于各温度下实际生成长度不同，不能简单归因于温度本身。结论仅适用于本轮固定 seed 的样本。

## 4. 人工评分

每条回答按 0 至 4 分评分：

| 分数 | 含义 |
| ---: | :--- |
| 4 | 正确、完整，并遵守题目格式 |
| 3 | 基本正确，存在轻微遗漏或表述问题 |
| 2 | 有可用内容，但不完整或包含重要错误 |
| 1 | 存在严重概念错误 |
| 0 | 未回答或基本不可用 |

| Prompt 类别 | temp=0.2 | temp=0.5 | temp=0.8 | 主要观察 |
| :--- | ---: | ---: | ---: | :--- |
| 中文问答 | 2 | 2 | 2 | 三组都能说明优缺点，但都因 256 token 上限截断 |
| 三点摘要 | 4 | 4 | 4 | 三组均严格输出三点，完整准确 |
| Rust 代码解释 | 2 | 3 | 3 | 0.2 错把同步 `read_to_string` 误述为”异步操作”；0.5/0.8 解释正确但截断 |
| RPC 原因分析 | 3 | 3 | 3 | 三组均覆盖网络、序列化和同步开销，但都截断 |
| OS 知识题 | 2 | 2 | 1 | 0.2/0.5 基本正确但截断；**0.8 出现严重幻觉**（”零页映射”、”Gin 模型格式”等编造概念） |
| **合计** | **13/20** | **14/20** | **13/20** | **0.5 综合质量最优** |

评分只用于比较本轮三种配置，不代表通用模型能力排名。

## 5. 结果分析

### 5.1 温度对质量的影响

1. **温度不会修复知识错误**：三组在摘要和一般系统分析上表现稳定，但 OS 题仍存在
   `mmap`、页缓存和 GGUF 概念混淆，说明事实准确性需要人工复核或参考资料约束。
2. **中等温度（0.5）综合质量最优**：0.5 组避免了 0.2 的"异步"误述和 0.8 的"零页映射"
   等严重幻觉，在准确性和表达完整性之间取得了更好平衡。
3. **高温增加幻觉风险**：0.8 组的 Rust 解释更清晰，但 OS 题出现了"零页映射"、
   "Gin 模型格式"等编造概念，高温的随机性导致事实错误增加。
4. **输出上限影响评分**：中文问答在 256 token 上限处截断，三组都未完整展开云端推理，
   说明质量实验必须同时固定并记录 `max_tokens`。
5. **性能差异不是质量差异**：0.5 组平均速度最快（32.12 t/s），但人工总分仅微高。
   不能只用 tokens/s 判断输出质量。

### 5.2 部署配置对质量的影响

作为对比，我们验证了 `threads=4` 与 `threads=12` 对同一 prompt 的输出。在固定
`seed=42`、`temperature=0.2` 的条件下，两种配置生成的文本**完全一致**，仅端到端
耗时不同（threads=12 更快）。这说明部署层面的配置（线程数、batch size 等）主要影响
**推理性能**，而不改变**输出质量**；因此本报告重点对比采样参数 `temperature` 对质量的影响。

## 6. 结论

> 在本轮 Qwen3.5-2B Q4_K_M 实验中，三种温度配置的人工总分为：
> `temperature=0.2` 13/20，`temperature=0.5` **14/20**，`temperature=0.8` 13/20。
> **中等温度（0.5）综合质量最优**，既避免了低温的保守误述，也减少了高温的幻觉风险。
> 最终部署推荐 `temperature=0.5`，并对操作系统事实类回答增加校验。
