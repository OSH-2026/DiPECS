# Lab4 Qwen3.5 输出质量评估报告

## 1. 实验目标

使用固定的 5 条质量 prompt，对 Qwen3.5-2B Q4_K_M 在两种温度配置下的输出进行
对比，观察采样温度对响应速度、指令遵循、完整性和事实准确性的影响。

## 2. 实验配置

除温度外，两组配置保持一致：

| 项目 | 内容 |
| :--- | :--- |
| 模型 | Qwen3.5-2B Q4_K_M |
| 后端 | llama.cpp CPU |
| 线程数 | 12 |
| 上下文 | 1024 |
| Batch | 64 |
| Seed | 42 |
| Reasoning | off |
| 对比变量 | `temperature=0.2`、`temperature=0.8` |

Prompt 覆盖中文问答、摘要、Rust 代码解释、RPC 原因分析和 OS 知识题共 5 类。

## 3. 性能结果

| 配置 | 成功数 | 平均端到端耗时 | 平均生成速度 |
| :--- | ---: | ---: | ---: |
| temperature=0.2 | 5/5 | **8,868.8 ms** | **31.58 t/s** |
| temperature=0.8 | 5/5 | 9,454.2 ms | 28.90 t/s |

本轮低温组平均耗时低约 6.2%，平均生成速度高约 9.3%。由于每条输出的实际停止位置
和生成长度可能不同，不能把全部差异归因于温度；结论仅适用于本轮固定 seed 的样本。

## 4. 人工评分

每条回答按 0 至 4 分评分：

| 分数 | 含义 |
| ---: | :--- |
| 4 | 正确、完整，并遵守题目格式 |
| 3 | 基本正确，存在轻微遗漏或表述问题 |
| 2 | 有可用内容，但不完整或包含重要错误 |
| 1 | 存在严重概念错误 |
| 0 | 未回答或基本不可用 |

| Prompt 类别 | temp=0.2 | temp=0.8 | 主要观察 |
| :--- | ---: | ---: | :--- |
| 中文问答 | 2 | 2 | 两组都能说明本地推理优缺点，但达到 token 上限后云端部分不完整 |
| 三点摘要 | 4 | 4 | 均严格输出三点，信息覆盖完整 |
| Rust 代码解释 | 2 | 3 | 0.2 错把同步 `read_to_string` 描述成异步操作；0.8 对 `Result` 和 `?` 更准确 |
| RPC 原因分析 | 3 | 3 | 均覆盖网络、序列化和同步开销，但存在把模型权重视为每请求传输的过度概括 |
| OS 知识题 | 2 | 1 | 两组都提到 mmap、页缓存和缺页；0.8 出现错误 GGUF 展开和“零页映射”等严重误述 |
| **合计** | **13/20** | **13/20** | 本轮没有明确质量胜者 |

评分只用于比较本轮两种配置，不代表通用模型能力排名。

## 5. 结果分析

1. **温度不会修复知识错误**：两组在摘要和一般系统分析上表现稳定，但 OS 题仍出现
   `mmap`、页缓存和 GGUF 概念混淆，说明事实准确性需要人工复核或参考资料约束。
2. **高温提高了部分表达完整性，但也增加幻觉风险**：0.8 组的 Rust 解释更清晰，
   同时 OS 题出现了更严重的术语误述。
3. **输出上限影响评分**：中文问答在 256 token 上限处截断，两组都未完整展开云端推理，
   说明质量实验必须同时固定并记录 `max_tokens`。
4. **性能差异不是质量差异**：0.2 组更快，但人工总分相同。不能只用 tokens/s 判断
   输出质量。

## 6. 结论

> 在本轮 Qwen3.5-2B Q4_K_M 实验中，`temperature=0.2` 和 `temperature=0.8`
> 的人工总分均为 13/20。低温配置速度更快、表达更稳定；高温配置在 Rust 解释上更好，
> 但 OS 题幻觉更明显。最终部署应优先使用低温配置，并对操作系统事实类回答增加校验。
