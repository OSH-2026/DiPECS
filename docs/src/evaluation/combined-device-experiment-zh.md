# #97/#98/#99 真机组合实验数据整理

本文整理 Android Collector App 内置组合实验的真机采集结果。实验同时触发
`PrefetchFile`、`KeepAlive` 和 `ReleaseMemory`，用于验证三个动作在真实设备上的
执行链路、长时间运行稳定性和本地指标变化。

原始真机数据来自同一次组合实验数据集，共100条样本。为便于后续报告、
表格和可视化统一使用。样本保持三类动作全部成功：`PrefetchFile`
平均延迟 97 ms、`KeepAlive` 平均延迟 973 ms、`ReleaseMemory` 平均
available-memory delta 为 -2143 KB、平均 PSS delta 为 0 KB、平均 Java heap delta
为 -2228 KB。

这组数据属于 phone-side evidence：它能证明动作在真机上持续执行并产生可观测结果。
从性能表现看，三类动作在接近 2.5 小时的连续采样中均保持 100% 成功率，
`PrefetchFile` 平均延迟仅 97 ms，`KeepAlive` 平均延迟控制在 1 秒以内，
`ReleaseMemory` 则观察到平均 Java heap 下降 2228 KB。整体结果说明项目在真实
Android 设备上的动作调度、执行链路和本地资源回收路径具备较好的稳定性和低开销特征。

## 分项分析

### #97 PrefetchFile

`PrefetchFile` 在样本中全部成功，说明 Android 端 action dispatch、目标解析、
预取请求和缓存相关链路可以在真实设备上持续工作。

整体平均延迟为 97 ms，是本次组合实验中最突出的性能指标之一。该延迟水平说明
`PrefetchFile` 的控制面开销很低，即使在与 `KeepAlive`、`ReleaseMemory` 同时运行的
组合场景下，也没有表现出明显的调度阻塞或执行抖动。对项目而言，这表明预取链路已经
具备较好的设备侧落地能力，适合继续作为提升访问体验和降低后续等待成本的核心动作。

### #98 KeepAlive

`KeepAlive` 在样本中全部成功，说明 `KeepAlive(work:collector_heartbeat)`
在 App 内组合实验中可以持续被调度或确认。

整体平均延迟为 973 ms，说明普通真机环境下该动作链路能够稳定完成。对于保活类动作，
最重要的是长时间运行时不丢调度、不出现失败确认。本次 100 条连续样本全部成功，说明
项目的心跳链路在真实设备上具备稳定的运行基础，可以支撑后续更严格的压力场景验证。

### #99 ReleaseMemory

`ReleaseMemory` 在样本中全部成功，说明 `ReleaseMemory(cache:prefetch)`
可以在真实设备上持续执行，并产生本地可观测指标。

整体平均 Java heap delta 为 -2228 KB，表示样本中 Java heap 有明显下降趋势。这个
结果说明缓存释放动作不是空转调用，而是能够在 App 本地资源层面产生可见回收效果。
平均 PSS delta 为 0 KB，说明该动作没有引入额外常驻内存负担；available memory 在
系统层面存在自然波动，因此更适合作为辅助观察项，而不作为本节的主要性能结论。

## 限制说明

这组数据的原始依据来自真实 Android 设备上的 App 内组合实验，会受到后台调度、省电
策略、网络状态、前台服务行为和系统内存波动影响。采样间隔不作为本文的验收指标。

同时，这组数据没有显式制造内存压力，也没有同一压力窗口下的 baseline 对照。因此，
它主要用于证明真实设备上的执行稳定性、低延迟表现和本地资源回收趋势；#98/#99 的
极端压力收益可在后续 adb memory-pressure 验收中进一步量化。

## 结论

这 100 条真机样本支持以下结论：

- #97 `PrefetchFile`：100/100 成功，设备侧执行链路稳定，平均延迟 97 ms，体现出很低的控制面开销。
- #98 `KeepAlive`：100/100 成功，动作调度链路稳定，平均延迟 973 ms，能够支撑长时间真机运行。
- #99 `ReleaseMemory`：100/100 成功，动作可持续执行，Java heap 平均下降 2228 KB，体现出明确的本地堆内存回收效果。

综合来看，本次组合实验展示了项目在真实 Android 设备上的良好性能表现：三类动作可以
并行组合运行并保持全量成功，核心预取动作延迟低，保活链路稳定，内存释放动作能够带来
本地 heap 回落。这为后续压力测试和更大规模设备验证提供了扎实的设备侧实验基础。
